# 认证问题 - 故障排除

如果在手动接收令牌时遇到问题，或者在使用资产接收令牌时看到以下错误：

> **SocketException: An attempt was made to access a socket in a way forbidden by its access permissions**

可以使用一位资产用户建议的解决方案。

<Aside type="caution">
⚠️ 注意：资产的作者未测试此解决方案，不对其使用后果负责。
</Aside>

---

## 解决步骤 (Windows)

### 第一步
以管理员身份打开 CMD.exe，输入以下命令，然后按 Enter：

```cmd
net stop winnat
```

### 第二步
输入以下命令，然后按 Enter：

```cmd
netsh int ipv4 add excludedportrange protocol=tcp startport=1923 numberofports=1
```

### 第三步
输入以下命令，然后按 Enter：

```cmd
net start winnat
```

### 第四步
重新尝试资产中的身份验证。

---

## 问题原因

此问题通常是由于 Windows NAT 驱动程序的端口限制导致的。某些防火墙或安全软件可能会阻止应用程序使用特定的端口范围。

## 替代解决方案

如果上述方法不起作用，您可以尝试：

1. **使用 Token 认证方式** - 不需要通过本地服务器接收令牌
2. **暂时禁用防火墙** - 测试是否是防火墙问题
3. **联系技术支持** - 如果问题持续存在
