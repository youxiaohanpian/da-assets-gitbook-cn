# 使用方法（UI Toolkit）

<Steps>

1. 在开始从 Figma 导入到 UI Toolkit 之前，你需要安装主资源 "**Figma Converter to Unity**"。
   要安装 "**Figma Converter to Unity**"，请按照 "[**开发者手册**](/69-fcu-for-developers-introduction/)" 中的说明进行操作。

2. 如果你想将你的 **UITK** 元素链接到 **MonoBehaviour** 脚本，还要安装免费的 "**UITK Element Linker**" 资源。
   你可以在 [**GitHub**](https://github.com/DA-Assets/UITK-Element-Linker) 下载此资源，或者如果在阅读此手册时资源已在 Asset Store 上发布，也可以在 [**Asset Store**](https://assetstore.unity.com/packages/tools/gui/282751) 下载。

3. 安装 "**Figma Converter to Unity**" 后，通过 Unity Package Manager 导入 "**Figma to UI Toolkit Converter**"。

4. 脚本重新编译后，前往 "**Figma Converter to Unity**" 资源设置，然后进入 **Main Settings** 选项卡，将 "**UI Framework**" 切换到 "**UITK**"。

   （图片：UI Framework 设置界面）

5. 如果你使用的是 "**UITK Element Linker**"，可以指定链接的类型。
   有关链接类型的更多信息，请参阅 "**UITK Element Linker**" 资源的手册。

   如果你指定 "**NONE**"，不会为你的布局创建任何用于链接的 Game 对象，也不会生成任何 C# 脚本。

   （图片：链接类型选择界面）

6. 之后，按照与 "**Figma Converter to Unity**" 资源相同的方式执行导入过程。
   更多详细信息请参阅 "开发者手册" — "[**导入帧**](/71-fcu-for-developers-import-frames/)" 部分。

7. 当你看到控制台中紫色的消息 "**Import Completed**" 后，你会看到：

   1. 在 "**UI Toolkit Output**" 文件夹中创建的 UXML 和 USS 文件。
   2. 链接到你的 UITK 元素的 Game 对象。

   （图片：UITK 导入结果）

</Steps>
